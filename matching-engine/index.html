Event-Driven Order Matching Engine

Overview

Motivation: This project was inspired by a topic I learned about in IE 421 - High Frequency Trading Technology at the University of Illinois at Urbana-Champaign.

Current Tech Stack: C++

The end goal for this project is to create a web app with 3 instances of the engine. I have already implemented the matching engine in C++ and I will build the web app using Streamlit. This means I will have to create a Python module that calls the C++ code using Python bindings, which is the next step of this project.

Source Code: Git Repository

NOTE: There is no demo video yet, because this project is not complete yet.

Matching Engine Features

  • Multiple Order Types - The engine currently supports market orders, limit orders, stop orders, stop-limit orders, and Fill-Or-Kill market orders.
  • Summary of Transaction - The engine also provides a summary of each transaction.
  • Order Visibility - The engine supports orders with "public" or "hidden" visibility and correctly sorts them in the order book.

Next Steps

The immediate next steps of this project are:

  1. Turn the C++ class into a Python module via Python bindings.
  2. Implement the Python module into a web app via Streamlit.
  3. Implement a database (potentially MongoDB?) into the web app, so all users can see and make orders to the same order book.
  4. Deploy the web app on the Streamlit Community Cloud to save on costs.
